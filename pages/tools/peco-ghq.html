「選ぶ」の peco × 「置く」の ghq

peco × ghq 実践ガイド

「あのリポジトリ、どこに clone したっけ？」「さっき打った長いコマンド、もう一回…」——その数秒の迷いを毎回ゼロにする2つの定番ツール。clone 先を規約で固定する ghq と、何でもインクリメンタル絞り込みできる peco を、セットアップから zsh 連携レシピまで一本にまとめました。

対象 初学〜実務 前提 ターミナルと Git の基本 形式 コマンド主体 読了 約 30 分
01

peco と ghq とは — 何が嬉しいのか

一言でいうと

ghq が「リポジトリを決まった場所に置く」係、peco が「一覧から1つを高速に選ぶ」係。2つを繋ぐと「リポジトリ名を数文字打って Enter で cd 完了」になる。

ghq（x-motemen/ghq）は、Git などのリモートリポジトリを ルート/ホスト名/オーナー名/リポジトリ名 という規約のディレクトリに clone してくれる管理ツール。「どこに clone するか」を毎回考えなくてよくなり、ghq list で手元の全リポジトリを機械可読な一覧として取り出せます。

peco（peco/peco）は、標準入力で受け取った行のリストをインクリメンタルに絞り込んで選び、選んだ行を標準出力に返すだけのシンプルなツール。「だけ」なのが強みで、パイプで繋げばコマンド履歴・ブランチ・プロセス・SSH先、何でも選択UIになります。

Why — なぜ2つセットなのか

ghq 単体では「整然と置ける」だけ、peco 単体では「選べる」だけ。ghq が選びやすい一覧を供給し、peco が選んだ結果を次のコマンドへ渡すことで初めて「数文字 → Enter → 移動完了」という体験になります。UNIX の「小さな道具をパイプで繋ぐ」思想のお手本のような組み合わせです。

02

セットアップ

一言でいうと

インストールは brew install peco ghq の一発。あとは ghq の置き場所（ルート）を決めるだけ。

macOS（Homebrew）/ Go 環境がある場合

$ brew install peco ghq
# Go でビルドして入れる場合（どの OS でも）
$ go install github.com/peco/peco/cmd/peco@latest
$ go install github.com/x-motemen/ghq@latest

次に、ghq がリポジトリを置くルートディレクトリを決めます。デフォルトは ~/ghq。変えたい場合は git の設定として書きます。

# 置き場所を ~/src にしたい場合（好みで。デフォルトの ~/ghq のままでも良い）
$ git config --global ghq.root '~/src'

# 現在のルートを確認
$ ghq root
/Users/you/src

動作確認まで一気にやってしまいましょう。

# 何か clone してみる
$ ghq get x-motemen/ghq
# 一覧に出れば OK
$ ghq list
github.com/x-motemen/ghq
# peco の動作確認：ls の結果を選択 UI にしてみる
$ ls | peco
Tip — ルートは最初に決め切る

ghq のルートを後から変えると、既存リポジトリの物理的な引っ越しが必要になります。~/ghq（デフォルト）か ~/src あたりを最初に決めて、以後は触らないのがおすすめです。

03

ghq の基本 — 置き場所を規約に任せる

一言でいうと

ghq get URL で clone すると、必ず ルート/ホスト/オーナー/リポジトリ に置かれる。人間は置き場所を考えるのをやめてよい。

clone する — ghq get

# URL 指定（https でも ssh でも OK）
$ ghq get https://github.com/x-motemen/ghq
$ ghq get git@github.com:peco/peco.git

# GitHub なら owner/repo の短縮形で書ける
$ ghq get x-motemen/ghq

# よく使うオプション
$ ghq get -u x-motemen/ghq      # clone 済みなら pull で更新
$ ghq get -p x-motemen/ghq      # ssh プロトコルで clone
$ ghq get --shallow torvalds/linux  # 浅い clone（巨大リポジトリ向け）

一覧する — ghq list

# 相対パス（ルートから）の一覧
$ ghq list
github.com/peco/peco
github.com/x-motemen/ghq

# フルパスで出す（cd やエディタに渡すならこちら）
$ ghq list -p
/Users/you/ghq/github.com/peco/peco
/Users/you/ghq/github.com/x-motemen/ghq

# 引数で部分一致フィルタ。-e で完全一致
$ ghq list peco
$ ghq list -p -e ghq

その他のサブコマンド

コマンド役割メモ
ghq rootルートのパスを表示スクリプトから参照するのに便利
ghq create owner/repo規約の位置に新規ローカルリポジトリを作る自作プロジェクトの始点も規約に乗せられる
ghq rm owner/repo管理下のリポジトリを削除確認プロンプトあり。それでも実行は慎重に
Point — 「一覧が機械可読」が本体

ghq の価値は clone の手間削減そのものより、手元の全リポジトリを ghq list -p 一発で行のリストとして取り出せることにあります。この出力が次節の peco の入力になります。

注意 — ghq 管理外で clone しない

一部のリポジトリだけ手で git clone してバラバラの場所に置くと、「ghq の一覧に出ないリポジトリ」が生まれて道具の信頼性が下がります。clone は常に ghq get、を習慣にするのがコツです。既存リポジトリの引っ越し方は FAQ（09）で扱います。

04

peco の基本 — パイプで「選ぶ」を差し込む

一言でいうと

何かの一覧 | peco | 次のコマンド。peco は行を受けて、選ばれた行を返す「選択フィルタ」。

peco を起動すると画面が選択 UI に切り替わり、上部の QUERY> に文字を打つたびに候補がリアルタイムで絞り込まれます。Enter で確定すると、選んだ行がそのまま標準出力に出る——それだけです。

# ファイルを選んでエディタで開く
$ vim $(find . -name '*.go' | peco)

# 引数にファイルを渡せば、その内容から選ぶ
$ peco /var/log/system.log

# 起動時にクエリを入れておく / プロンプト文字列を変える
$ ghq list -p | peco --query "shoka" --prompt "repo>"

キー操作

キー動作
↑ / ↓（C-p / C-n）候補の上下移動
Enter確定（選択行を標準出力へ）
C-c / Escキャンセル（何も出力せず終了コード ≠ 0）
C-Space複数選択のトグル（選んで次へ）。確定で全選択行が出力される
C-a / C-eクエリ行の先頭 / 末尾へ
C-u / C-wクエリをまとめて消す / 直前の単語を消す
C-rフィルタ方式の切り替え（次節）
Tip — スペース区切りは AND 検索

クエリに api user のようにスペース区切りで複数語を打つと、すべての語を含む行に絞り込まれます。長いパスやコマンド履歴から探すときに効きます。

Point — キャンセルは「失敗」として返る

C-c でキャンセルすると peco は何も出力しません。vim $(... | peco) のような使い方では、後続のコマンドに空文字が渡らないようシェル関数側で空チェックを入れるのが定石です（06 のレシピはすべて対応済み）。

05

peco を育てる — フィルタとオプションと設定

一言でいうと

絞り込み方式（IgnoreCase / SmartCase / Regexp / Fuzzy）は C-r で切替え。挙動の好みは ~/.config/peco/config.json に書く。

フィルタ方式

フィルタ挙動
IgnoreCase大文字小文字を無視した部分一致（デフォルト）
CaseSensitive大文字小文字を区別する部分一致
SmartCaseクエリが小文字だけなら無視、大文字を含むと区別
Regexp正規表現でマッチ
Fuzzyあいまいマッチ（gcg で git-complete-guide に当たる、fzf 風）

起動時にフィルタを指定する

$ ghq list -p | peco --initial-filter Fuzzy

知っておくと便利なオプション

  • --select-1 … 候補が最初から1件なら UI を出さず即確定。スクリプトが滑らかになる。
  • --initial-index N … 最初に選択しておく行を指定。
  • --on-cancel error … キャンセル時に終了コードを明示的にエラーへ。
  • --rcfile PATH … 設定ファイルを指定して起動。

設定ファイル

キーバインドや配色は ~/.config/peco/config.json でカスタマイズできます。たとえば既定フィルタを Fuzzy 寄りに育てたり、選択キーを変えたり。

// ~/.config/peco/config.json（例）
{
  "InitialFilter": "SmartCase",
  "Keymap": {
    "C-j": "peco.SelectDown",
    "C-k": "peco.SelectUp"
  }
}
注意 — 設定はまず素のまま使ってから

peco はデフォルトで十分使いやすく設計されています。最初から設定を盛るより、不便を感じた箇所だけ最小限に変える方が、他人の環境や ssh 先でも手が迷いません。

06

黄金コンビ — リポジトリへ瞬間移動する

一言でいうと

ghq list -p | peco の結果へ cd する関数をキーに割り当てる。Ctrl-] 一発で全リポジトリがジャンプ先になる。

ここが本ガイドの山場です。zsh なら ~/.zshrc に次を追記してください。

~/.zshrc — Ctrl-] でリポジトリ選択 → cd

function peco-src() {
  local selected
  selected=$(ghq list -p | peco --query "$LBUFFER")
  if [ -n "$selected" ]; then
    BUFFER="cd ${selected}"
    zle accept-line
  fi
  zle clear-screen
}
zle -N peco-src
bindkey '^]' peco-src

読み込み直して（source ~/.zshrc）、Ctrl-] を押すと全リポジトリの一覧が開きます。数文字打って Enter——これで どこにいても任意のリポジトリへ2秒で移動できます。

bash 派はこちら（~/.bashrc）

peco-src() {
  local selected
  selected=$(ghq list -p | peco --query "$READLINE_LINE")
  if [ -n "$selected" ]; then
    READLINE_LINE="cd ${selected}"
    READLINE_POINT=${#READLINE_LINE}
  fi
}
bind -x '"\C-]": peco-src'
Why — なぜ alias でなく zle ウィジェットなのか

ただの関数や alias でも動きますが、zsh の zle（ラインエディタ）ウィジェットにすると入力中のコマンドラインを壊さずに割り込めるうえ、$LBUFFER（入力済み文字列）を初期クエリとして引き継げます。「途中まで打った名前で絞り込み始める」自然な体験になります。

Point — 新しいリポジトリも増えた瞬間から対象

一覧は毎回 ghq list -p を実行して作るので、ghq get した直後のリポジトリも即座にジャンプ先に含まれます。キャッシュ更新のような儀式は不要です。

07

活用レシピ集 — 「一覧 | peco | コマンド」を量産する

一言でいうと

パターンはすべて同じ：一覧を出す → peco で選ぶ → 結果を次に渡す。自分の「よく探すもの」に当てはめれば無限に作れる。

① コマンド履歴を検索（Ctrl-R を置き換え）

~/.zshrc — 履歴をインクリメンタル検索して呼び戻す

function peco-history() {
  BUFFER=$(history -n -r 1 | peco --query "$LBUFFER")
  CURSOR=$#BUFFER
}
zle -N peco-history
bindkey '^r' peco-history

標準の Ctrl-R（1件ずつ遡る）と違い、候補一覧を見ながら AND 検索で探せます。「先週打った長い docker コマンド」も数文字で戻ってきます。

② ブランチを選んで切り替え

$ git switch $(git branch --format='%(refname:short)' | peco)

# 関数にしておくと楽（gsw と打つだけ）
function gsw() {
  local b
  b=$(git branch --format='%(refname:short)' | peco)
  [ -n "$b" ] && git switch "$b"
}

③ 選んだリポジトリをエディタで開く

# VS Code で開く
$ code $(ghq list -p | peco)

# GitHub のページをブラウザで開く（macOS）
$ open "https://$(ghq list | peco)"

④ SSH 接続先を選ぶ

$ ssh $(grep -E '^Host ' ~/.ssh/config | awk '{print $2}' | grep -v '\*' | peco)

⑤ プロセスを選んで kill（複数選択の出番）

# C-Space で複数選んでまとめて kill できる
$ ps aux | peco | awk '{print $2}' | xargs kill

⑥ 変更ファイルを選んで git add

# これも C-Space の複数選択が活きる
$ git add $(git status -s | peco | awk '{print $2}')
Danger — 破壊的コマンドに直結させない

kill や git checkout -- 、rm のような取り返しのつかないコマンドへ peco の出力を直結するときは、選択ミス＝即実行になることを忘れずに。まずは echo を挟んで出力を確認してから繋ぐ癖をつけると安全です。

Tip — レシピの作り方は3問だけ

「①何の一覧か（コマンドで出せるか）」「②1行に必要な情報が入っているか」「③選んだ行をどう加工して渡すか（awk/cut）」。この3つに答えられれば、それはもう peco レシピになります。

08

fzf との比較・使い分け

一言でいうと

役割はほぼ同じ「選択フィルタ」。fzf は高機能（あいまい検索既定・プレビュー・補完統合）、peco は小さく素直。どちらか一方で十分で、ghq とはどちらも組める。

観点pecofzf
既定の検索部分一致（Fuzzy へ切替可）あいまい（fuzzy）検索
プレビュー表示なし--preview でファイル内容等を横に表示
シェル統合自分で関数を書く（本ガイドの形）同梱の keybinding / 補完スクリプトが豊富
設計小さく単機能。設定なしでも完結多機能。オプションが非常に多い
共通点標準入力 → 選択 → 標準出力、という同じインタフェース。レシピは互いにほぼ移植可能
Point — 思想で選べばよい

「道具は薄く、組み立ては自分で」が好きなら peco、「全部入りの便利さ」が好きなら fzf。本ガイドのレシピは peco を fzf に置き換えてもほぼそのまま動くので、移行コストを心配する必要はありません。

09

つまずきポイント FAQ

Q1. すでに手で clone したリポジトリが散らばっている

同じリポジトリをもう一度 ghq get し直すのが最も簡単で確実です（リモートが真実なので、ローカルの未 push 変更だけ退避してから）。物理移動したい場合は、ghq root 配下の規約どおりの位置（ルート/ホスト/オーナー/リポジトリ）へ mv すれば ghq は認識します。

# 例：~/work/myrepo を規約の位置へ移す
$ mkdir -p $(ghq root)/github.com/your-name
$ mv ~/work/myrepo $(ghq root)/github.com/your-name/myrepo
$ ghq list myrepo   # 出てくれば認識されている

Q2. パスが深くて手で cd するのが辛い

それは仕様であり、手で cd しなくなるのが前提の設計です。06 の Ctrl-] を入れた瞬間に深さは問題でなくなります。むしろ深い規約パスのおかげで衝突や迷子が起きません。

Q3. 会社の GitLab や自前 Git サーバでも使える？

使えます。ghq get https://gitlab.example.com/team/repo のように URL で渡せば、ルート/gitlab.example.com/team/repo に置かれます。ホスト名でディレクトリが分かれるので、GitHub と社内リポジトリが混ざることもありません。

Q4. peco の画面でキャンセルしたのにコマンドが走った

$(... | peco) を直接埋め込むと、キャンセル時に空文字で後続が実行されることがあります。レシピのように一度変数に受けて [ -n "$selected" ] で空チェックする形にしてください。

Q5. リポジトリが増えて一覧が重くなった

ghq list はディレクトリ走査なので通常は一瞬です。気になる場合は ghq list 部分文字列 で先に絞ってから peco に渡す、巨大リポジトリは --shallow で取る、などで快適さを保てます。

用語集

インクリメンタル検索
1文字打つごとに候補がリアルタイムで絞り込まれていく検索方式。peco の基本動作。
ghq root（ルート）
ghq がリポジトリを配置する起点ディレクトリ。デフォルトは ~/ghq、git config ghq.root で変更できる。
ディレクトリ規約
ルート/ホスト名/オーナー名/リポジトリ名 という ghq の配置ルール。URL の構造をそのままパスに写す。
フィルタ（peco）
クエリと候補行の照合方式。IgnoreCase / CaseSensitive / SmartCase / Regexp / Fuzzy があり C-r で切替。
zle ウィジェット
zsh のラインエディタに登録する自作機能。zle -N 関数名 で登録し bindkey でキーに割り当てる。
$LBUFFER
zsh で、カーソルより左側の入力済み文字列。peco の初期クエリに渡すと「打ちかけの文字で絞り込み開始」できる。
fzf
peco と同役割の選択フィルタ。あいまい検索が既定で、プレビューやシェル統合が豊富。
